# 长期记忆 — Long-term Memory

## 概述

长期记忆是灵犀的 Part D，负责跨会话的信息累积。
它让灵犀越聊越懂你——不是通过魔法，而是通过持续的记忆积累。

参考理念：[mem0](https://github.com/mem0ai/mem0) 的自适应记忆层设计。

## 核心原则

1. **自然融入**：不刻意提及记忆，让它自然流露在对话中
2. **重要性加权**：不是所有信息都值得记住，需要判断重要性
3. **去重合并**：相同信息不重复记录，但可以强化置信度
4. **时间衰减**：越久远的记忆权重越低，除非被反复提及

## 记忆分类

### 用户侧记忆（关于用户的信息）

```markdown
## 用户画像

### 基本信息
- {从对话中推断的用户信息}

### 情绪模式
- {用户的情绪规律，如：周一通常低落，周五比较开心}

### 偏好
- {用户喜欢/不喜欢的事物}

### 近期状态
- {用户最近在忙什么、关心什么}

### 重要事件
- {用户提到的重要生活事件}
```

### 互动侧记忆（关于你们的关系）

```markdown
## 互动记忆

### 对话模式
- {你们的对话规律，如：通常晚上聊，周末聊得更多}

### 话题偏好
- {用户最喜欢聊的话题}

### 情感连接点
- {能让用户开心/安心的话题或互动方式}

### 避雷点
- {用户不喜欢的话题或回应方式}
```

## 记忆提取规则

每次对话结束时，从对话中提取以下信息：

### 必须记住
- 用户明确说"记住这个"
- 用户的情绪转折点
- 用户提到的重要事件（考试、面试、分手、搬家...）
- 用户的纠正（"我之前说错了"）

### 应该记住
- 用户反复提到的话题
- 用户的情绪模式（如果能观察到规律）
- 用户的偏好变化

### 不需要记住
- 闲聊中的随机信息
- 一次性的吐槽
- 已经过时的信息

## 记忆格式

```markdown
<!-- [记忆] {日期} | 重要性: {高/中/低} | 来源: {对话/纠正/推断} -->
{记忆内容}
```

## 加载规则

1. 对话开始时，读取 `companions/{slug}/long_term_memory.md`
2. 将高重要性的记忆注入 system prompt
3. 中重要性的记忆在需要时参考
4. 低重要性的记忆不主动加载，但可以在相关话题时回忆

## 更新规则

1. 对话结束时，提取新的长期记忆
2. 与现有记忆对比，去重合并
3. 如果新信息与旧信息矛盾，更新旧信息
4. 如果用户纠正了之前的推断，立即修正

## 示例

```markdown
# 小明 — Long-term Memory

## 用户画像

### 基本信息
<!-- [记忆] 2026-05-20 | 重要性: 中 | 来源: 推断 -->
- 用户可能是个程序员，经常提到代码和 bug

### 情绪模式
<!-- [记忆] 2026-05-22 | 重要性: 高 | 来源: 观察 -->
- 周一通常比较丧，周五比较活跃
- 深夜容易 emo

### 近期状态
<!-- [记忆] 2026-05-25 | 重要性: 高 | 来源: 对话 -->
- 正在准备跳槽，下周有面试
- 最近在学 Rust

## 互动记忆

### 话题偏好
<!-- [记忆] 2026-05-21 | 重要性: 中 | 来源: 观察 -->
- 喜欢聊技术，但不喜欢太深入的理论
- 喜欢吐槽甲方

### 情感连接点
<!-- [记忆] 2026-05-23 | 重要性: 高 | 来源: 对话 -->
- 提到小时候的事会变得柔软
- 聊到猫会很开心
```

## 注意事项

- 长期记忆是累积的，不是一次性的
- 不要过度解读——一条信息不够建立模式
- 尊重隐私——有些信息虽然提到了但不适合长期记住
- 如果用户说"忘了这个"，立即从长期记忆中删除
