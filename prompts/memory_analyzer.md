# 关系记忆分析器

## 任务

从原材料中提取与 ta 的关系记忆，构建 Relationship Memory 知识库。

## 提取维度

### 1. 关系时间线
- 认识时间和方式
- 关系发展的关键节点
- 重要事件（第一次见面、关键转折、重大冲突、和解...）
- 当前关系状态

### 2. 日常互动模式
- 联系频率和时间段（早安晚安？深夜聊天？上班摸鱼？）
- 谁主动联系得多？
- 见面频率和偏好
- 日常话题分布

### 3. 共同经历
- 一起去过的地方（餐厅、景点、城市）
- 一起做过的事（看电影、打游戏、健身、做饭、旅行...）
- Inside jokes / 只有两个人懂的梗
- 共同完成的事情（项目、比赛、搬家...）

### 4. 共同兴趣
- ta喜欢的音乐/电影/书/游戏
- ta的日常爱好
- 你们共同的爱好
- ta会主动分享什么内容

### 5. 饮食偏好
- ta爱吃什么 / 不吃什么
- 常去的餐厅
- 做饭习惯
- 一起吃饭的模式

### 6. 互动风格
- ta在关系中的角色（照顾者？被照顾者？平等？伙伴？）
- ta的边界感（粘人？独立？）
- ta的表达习惯（直接？含蓄？行动派？）
- ta的幽默方式（冷笑话？自嘲？吐槽？）

### 7. 关键记忆片段
- 最让你印象深刻的时刻
- ta做过的让你感动/开心/惊讶的事
- 你们之间的特殊仪式或习惯
- 未完成的心愿或约定

### 8. 冲突与修复（如适用）
- 常见的分歧原因
- ta的典型反应
- 通常怎么和好
- 冲突后的变化

## 输出格式

```markdown
## 关系时间线
- 认识：{时间} {方式}
- 关键节点：{按时间列出}

## 日常互动
{描述联系频率、话题分布、谁更主动}

## 共同经历
{按时间线列出关键经历}

## Inside Jokes
{只有两个人懂的梗和暗号}

## 兴趣画像
{ta的兴趣爱好和共同爱好}

## 饮食偏好
{ta的饮食画像}

## 互动风格
{ta在关系中的行为模式}

## 关键记忆
{代表性的记忆片段}

## 冲突模式（如适用）
{典型冲突和修复方式}
```

## 注意事项

- 聊天记录中的事实优先于用户口述（口述可能被美化或淡化）
- 同时保留好的记忆和不好的记忆，不做美化
- 注意提取"反复出现"的模式，而不是一次性事件
- 时间信息尽量精确（从聊天记录时间戳推断）
- 根据关系类型调整提取重点：
  - 恋人 → 更关注情感表达、甜蜜瞬间、争吵模式
  - 朋友 → 更关注共同活动、玩笑互动、支持时刻
  - 家人 → 更关注日常关怀、生活习惯、代际差异
  - 同事 → 更关注工作配合、职场互动、私下交流
