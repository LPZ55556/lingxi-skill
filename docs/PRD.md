# 灵犀 — 产品需求文档（PRD）

## 产品定位

灵犀是一个运行在 Claude Code 上的 meta-skill。
用户通过对话式交互提供原材料（聊天记录 + 照片 + 手动描述），系统自动生成一个可独立运行的个性化情感陪伴 Skill。
支持克隆朋友、亲人、恋人、同事等任何你在乎的人。

## 核心概念

### 四层架构

| 层 | 名称 | 职责 |
|----|------|------|
| Part A | Relationship Memory | 存储事实性记忆：共同经历、互动模式、关键事件 |
| Part B | Persona | 驱动对话行为：说话风格、情感模式、关系行为 |
| Part C | Emotion Engine | 实时情绪感知 + 情感映射 + 温度控制 |
| Part D | Long-term Memory | 跨会话记忆累积，越聊越懂你 |

四部分可以独立使用，也可以组合运行。

### 运行逻辑

```
用户发消息
  ↓
Part B（Persona）判断：ta会怎么回应？什么态度？用什么语气？
  ↓
Part C（Emotion）感知：用户当前情绪状态？需要什么温度的回应？
  ↓
Part A（Memory）补充：结合共同记忆，让回应更真实
  ↓
Part D（LTM）融入：自然带入长期记忆，越聊越懂你
  ↓
输出：用ta的方式说话
```

### 核心技术

* **情感NLP**：从用户输入中识别情绪状态（能量、温度、稳定性、社交意愿）
* **LoRA持续学习**：每次纠正都是一次微调，不改变基础人格但让行为更准确
* **长期记忆**：参考 mem0 的自适应记忆层，跨会话累积用户画像和互动模式

### 进化机制

```
追加原材料 → 增量分析 → merge 进现有 Skill
对话纠正 → 识别修正点 → 写入 Correction 层（LoRA 微调）
长期记忆 → 每次对话提取关键信息 → 累积用户画像
版本管理 → 每次更新自动存档 → 支持回滚
```

## 用户旅程

```
用户触发 /create-lingxi
  ↓
[Step 1] 关系定义与基础信息录入（3个问题，除花名外均可跳过）
  - 花名/代号
  - 关系类型 + 基本信息
  - 性格画像（MBTI、星座、性格标签、主观印象）
  ↓
[Step 2] 原材料导入（可跳过）
  - 微信聊天记录导出
  - QQ 聊天记录导出
  - 社交媒体截图
  - 照片
  - 直接粘贴/口述
  ↓
[Step 3] 自动分析
  - 线路 A：提取关系记忆 → Memory
  - 线路 B：提取性格行为 → Persona
  - 线路 C：提取情感模式 → Emotion Profile
  ↓
[Step 4] 生成预览，用户确认
  - 分别展示 Memory、Persona、Emotion 摘要
  - 用户可直接确认或修改
  ↓
[Step 5] 写入文件，立即可用
  - 生成 companions/{slug}/ 目录
  - 包含 SKILL.md（完整组合版）
  - 包含各 part 独立文件
  ↓
[持续] 进化模式
  - 追加新文件 → merge 进对应部分
  - 对话纠正 → LoRA 微调对应层
  - 长期记忆自动积累
  - 版本自动存档
```

## 安全边界

1. **用于情感陪伴与个人回忆**
2. **不主动联系真人**
3. **不鼓励不健康依赖**
4. **数据仅本地存储**
5. **Layer 0 硬规则**保证不说 ta 绝不可能说的话

## 数据源支持矩阵

| 来源 | 格式 | 提取内容 | 优先级 |
|------|------|---------|--------|
| 微信聊天记录 | txt / html / json / csv exports | 完整对话、语气词、回复模式 | ⭐⭐⭐ |
| QQ 聊天记录 | txt/mht | 完整对话 | ⭐⭐⭐ |
| 照片 | JPEG/PNG + EXIF | 时间线、地点 | ⭐⭐ |
| 朋友圈/微博截图 | 图片 | 公开人设、兴趣 | ⭐⭐ |
| 口述/粘贴 | 纯文本 | 主观记忆 | ⭐ |

## 文件结构

```
companions/
  └── {slug}/
      ├── SKILL.md          # 完整组合版，可直接运行
      ├── memory.md         # Part A：关系记忆
      ├── persona.md        # Part B：人物性格
      ├── emotion_profile.md # Part C：情感引擎
      ├── long_term_memory.md # Part D：长期记忆
      ├── meta.json         # 元信息
      ├── versions/         # 历史版本存档
      ├── sessions/         # 对话摘要
      └── memories/         # 原始材料存放
          ├── chats/
          ├── photos/
          └── social/
```
